# Security Policy

## Supported Versions

This section outlines the project versions currently receiving security updates, ensuring users are aware of the supported versions for optimal security practices.

| Version | Supported  | Notes                                       |
| ------- | ---------- | ------------------------------------------- |
| 5.1.x   | ✅         | Actively supported with security patches.   |
| 5.0.x   | ❌         | No longer supported; users are encouraged to upgrade to 5.1.x or higher. |
| 4.0.x   | ✅         | Supported with critical security updates only. |
| < 4.0   | ❌         | No support; users must upgrade to at least 4.0.x for security compliance. |

## Reporting a Vulnerability

This section explains how to report a security vulnerability to ensure timely and effective mitigation.

To report a vulnerability, please follow these steps:

1. **Submission**: Submit your report to our dedicated security email at info(at)eliaslenz-mbaberatung.de
2. **Confirmation**: You will receive a confirmation email within 72 hours acknowledging receipt of your report.
3. **Investigation Timeline**: Expect an initial update within two weeks, detailing the status of the investigation. For more complex issues, the timeline may extend; we will keep you updated as progress is made.
4. **Resolution**: If the vulnerability is confirmed, we will provide details about the fix and the expected patch release timeline. If the report does not meet our criteria for a vulnerability, we will inform you of the decision and provide reasoning.

We take security seriously and appreciate your help in keeping the project safe. Thank you for your support!# Security Policy
